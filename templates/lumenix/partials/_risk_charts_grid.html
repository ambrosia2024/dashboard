Data assumptions – toxin chart

Data assumptions

  • Data is simulated for one crop–pathogen pair for the chosen period.
  • We start from a base toxin level of 8 μg/kg.
  • Summer and early autumn are made higher with a simple seasonal curve.
  • Each month gets a bit of random change to look more realistic.
  • Some months can get an extra spike if we mark an event (e.g. heavy rain).
  • We show one test/action limit of 12 μg/kg for every month.
  • Temperature and humidity in the tooltip are also simulated.

This chart is only to show how toxin levels could change over months in a demo setup.

How this chart is built

  • date is used on the X-axis.
  • toxin_level_ug_per_kg (μg/kg) is drawn as the main line.
  • toxin_limit_ug_per_kg (μg/kg) is shown as a horizontal “limit” line. If it is missing, the chart uses 12 μg/kg.
  • The tooltip also shows temperature_c, humidity_pct, and event (if present) to explain spikes.
  • The Y-axis is sized so the limit is always visible.
Data assumptions – pathogen chart

Data assumptions

  • Data is simulated for the same crop–pathogen pair and dates as the toxin chart.
  • Pathogen values are shown as generic units/g – not lab values.
  • We make late summer / autumn a bit higher to show seasonality.
  • Some months can get extra growth if an event is present.
  • The first month in the series is used as the baseline.
  • Point colours come from how much a month is above that baseline (small, moderate, large, significant). This is just to spot higher months quickly.
  • Other values shown with it are also simulated to match the story.

Use this chart to compare months with each other, not as exact measurements.

How this chart is built

  • date is used on the X-axis.
  • pathogen_conc_units_per_g (units/g, demo) is plotted as a line.
  • The first point in the series is treated as the baseline.
  • For every other point, the chart works out how far it is above that baseline and colours the point (small, moderate, large, significant).
  • The tooltip shows the current value, the baseline value, and the % change vs baseline.
Data assumptions – probability of illness

Data assumptions

  • Data is simulated from the same time series used for toxins and pathogens.
  • We treat the first month as the starting (baseline) probability.
  • Later months are shown above or below this starting value.
  • Probabilities come from the demo model logic (e.g. higher toxin → higher chance), not from clinical data.
  • We add small random changes so the line is not perfectly flat.
  • The colour bands are based on “how many times above baseline” and are only for quick reading.

This chart shows the shape of change over time, not real case risk.

How this chart is built

  • date is used on the X-axis.
  • prob_illness_pct (%) is the main line.
  • baseline_prob_illness_pct (%) is drawn as a dashed line if it is available; otherwise the chart uses the first point as the baseline.
  • A colour scale (visualMap) uses risk_multiplier to show when a month is clearly above baseline.
  • The tooltip shows: date, current probability, baseline, × baseline, and % difference.
  • Clicking on a point can pin it and show an explanation for that exact month.
Data assumptions – cases per 100k

Data assumptions

  • Data is simulated from the same monthly series as above.
  • Cases per 100k are calculated, not reported.
  • We assume about 40,000 people exposed per 100,000 in a month.
  • So: cases_per_100k ≈ (probability % / 100) × 40,000, then rounded.
  • We also show a short rolling average to smooth jumps between months.
  • This is to show trends in the demo data, not real surveillance numbers.

Read this chart as “if this model were true, this is the size of the bump”.

How this chart is built

  • date is used on the X-axis.
  • cases_per_100k is shown as bars.
  • A rolling mean is computed in JS using window.RISK_CONFIG.rollingWindow and drawn as a line.
  • cases_per_100k itself comes from the modelled probability (it is derived, not observed).
Data assumptions – seasonal heatmap

Data assumptions

  • The heatmap uses the same simulated monthly rows as the other charts.
  • Each cell is one month in one year (e.g. May 2023).
  • By default we colour by the risk multiplier (current ÷ baseline).
  • A value around 1 means “like baseline”; higher than 1 means “above baseline”.
  • The colour scale is fixed (green → orange → red) just to make differences clear.
  • Years shown are only the years present in the data.
  • All values are demo values, not monitoring data.

This view is for spotting seasonal/high months at a glance.

How this chart is built

  • The chart takes the same monthly rows and groups them by month (Jan–Dec) and year.
  • month is shown on the X-axis, year is shown on the Y-axis.
  • By default the value in each cell is risk_multiplier (× baseline).
  • If the mode is set to "prob_illness_pct", the value becomes the probability (%) instead.
  • A fixed colour scale (green → orange → red) is used so different years can be compared easily.
  • The tooltip shows the month/year and the exact value for that cell.
Data dictionary (synthetic series)

Units summary (quick reference)

  • date – ISO date
  • year – number (e.g. 2023)
  • month – 1–12
  • temperature_c – °C
  • humidity_pct – %
  • toxin_level_ug_per_kg – μg/kg
  • toxin_limit_ug_per_kg – μg/kg
  • pathogen_conc_units_per_g – units/g (demo)
  • prob_illness_pct – %
  • baseline_prob_illness_pct – %
  • risk_multiplier – × (dimensionless ratio)
  • cases_per_100k – cases / 100,000 people
  • event – string label

These fields are generated once per month for the selected crop–pathogen pair and then reused by all charts.
